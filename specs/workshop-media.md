# Workshop Media

## Purpose

Define how images and videos are used across workshop content to create a clear, professional learning experience. Media should help learners understand concepts — not serve as decoration.

## Image Hierarchy

Images can appear at five distinct content levels, each with a different role:

### Workshop Thumbnail

A branded banner image displayed on the workshop card on the Home and Workshop Overview pages. It visually represents the workshop's purpose and identity, helping learners recognize and distinguish workshops at a glance.

### Lesson Header

A key visual summarizing the lesson topic. It appears both on the lesson card in the overview grid and as a header when viewing the lesson in detail. This image sets the tone for the entire lesson.

### Section Image

A full-width image displayed above the section explanation. It provides context for the section content — either a screenshot showing a specific UI feature, or an illustration explaining a concept. Section images support click-to-zoom for closer inspection.

### Example Image

A small thumbnail displayed alongside a Q&A card. It provides a focused visual hint related to the specific question, such as a cropped screenshot of a UI element or a small icon representing the concept being asked about.

### Assessment Image

A small thumbnail displayed next to an assessment question. It offers a visual cue related to the question, helping learners connect the question to the relevant concept or feature.

## Videos in Sections

Sections can include embedded videos to demonstrate features, explain workflows, or provide supplementary visual content. Videos appear inline within the section and support:

- YouTube embeds, automatically converted to the proper embed format.
- Local video files hosted alongside the workshop content.

Videos and section images serve the same contextual role — when a video is present, it typically replaces the section image.

## Design Principles

### Screenshots for UI Context

Use screenshots only when explaining a specific screen, button, or interface element. Screenshots should show exactly what the learner will see, making instructions concrete and verifiable.

### Illustrations for Concepts

Use illustrations or diagrams when explaining ideas, workflows, or abstract features. Concept graphics should clarify relationships and processes that a screenshot cannot convey.

### Purposeful Media

Every image should help the learner understand something. Avoid purely decorative images that add visual noise without aiding comprehension.

### Visual Consistency

Maintain the same visual style across all workshops and languages. Consistent image sizing, formatting, and tone create a cohesive learning experience.

## Light and Dark Mode Compatibility

All media should work well in both light and dark themes. Use neutral backgrounds or transparency where possible so that images do not clash with the surrounding interface in either mode.

## Builtin Workshops as Showcase

The platform's builtin workshops (such as the Open Learn Guide and Feedback workshops) serve as a showcase for all media features. They demonstrate the full image hierarchy — from workshop thumbnails to assessment-level images — and model best practices for workshop creators to follow when building their own content.
